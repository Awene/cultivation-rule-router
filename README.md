# 规则路由 · Rule Router

一个 **SillyTavern 扩展**：在每次生成前，用一个**轻量（flash）副模型**根据当前情境，智能判断**该激活哪些世界书条目**，把无关条目本回合临时隐藏，只把相关条目交给主模型。

> 适用场景：你的世界书里有大量**常驻（蓝灯）条目**——规则、设定、状态模板等——它们每回合都被塞进提示词，既费 token 又干扰主模型。本插件让这些条目**按需激活**：该用时才用。

- **零改卡**：不修改你世界书条目的开关，不写盘。隐藏只作用于"本次扫描"。
- **通用**：任何角色卡都能用，逐条目在 UI 里配置触发条件。
- **安全降级**：未配置副模型 / 判定失败时，全部条目照常，不影响正常游玩。

---

## 工作原理

```
玩家发送 → GENERATION_AFTER_COMMANDS 拦截（跳过 dryRun/后台/代写生成）
  1. 收集候选 = 你在 UI 里开了「文字过滤」并填了条件的世界书条目
  2. flash 副模型据【最近剧情 + 本轮输入】逐条判断其条件是否满足 → 命中集
     （相同请求会命中本地结果缓存，直接复用、不重复调用）
  3. 命中条目 + 其关联条目 = 保留；其余候选 + 未触发的被动条目 = 本次隐藏
  4. WORLDINFO_ENTRIES_LOADED 时把"隐藏集"临时置 disable（仅本次扫描，不落盘、不动 UI）
→ ST 照常扫描 → 只有相关条目进入提示词（EJS/宏由 ST 原生处理）→ 主模型生成
```

机制经 SillyTavern 1.16 源码确认：`getSortedEntries` 在克隆条目前 emit `WORLDINFO_ENTRIES_LOADED`，此时改 `disable` 只影响本次扫描，不触碰存储与界面开关。

## 安装

**方式一（推荐，可自动更新）**：ST → 扩展 → 安装扩展 → 填入 git 地址：

```
https://github.com/Awene/cultivation-rule-router
```

国内连不上 GitHub 时改用镜像地址：`https://gitclone.com/github.com/Awene/cultivation-rule-router`

**方式二（手动）**：把本目录放到 `<SillyTavern>/data/<用户名>/extensions/cultivation-rule-router/`，刷新页面。

## 配置

魔棒菜单（左下角✨）→「**规则路由配置**」打开配置面板：

- **flash 路由模型**：填副模型的接口地址 / API Key，点「获取模型」选一个模型。建议用便宜快速的小模型（如 deepseek-flash、gemini-flash 等）。
- **路由提示词**：给副模型的 system 提示词，可编辑 + 「预览」查看副模型实际收到的完整提示词。
- **上下文设置**：
  - *思维链分隔符*（默认 `</think>`）：分隔符之前的内容视作思维链，不送入副模型。
  - *最近 GM 回复条数*：只收录最近 N 条 GM 回复作为情境（玩家历史输入不进，只保留本轮输入）。
  - *标签剔除*：把 GM 回复里 `<标签>…</标签>` 整段删掉再送入副模型（逗号分隔多个）。
  - *结果缓存条数*：请求相同（重生成/回退同输入）时复用旧判定、不重复调用副模型（0=关闭）。
- **规则过滤**：「刷新」列出当前使用中的世界书条目，逐条：
  - 勾选「文字过滤」并填**启用条件**（副模型据此判断本条该不该开）。
  - 「**关联**」：让本条目触发时**连带开启**其它条目（如 A→B：A 开则 B 必开）。被关联的条目变为**被动**（不能独立开启、只被关联触发），支持 A→B、C→B 多源。
- **配置导入 / 导出**（顶部图标）：可分享给别人；**导出绝不含 API Key**，导入保留本地 Key。
- **总开关**：一键关闭本插件（关闭后所有条目照常）。

## 逐层查看

每个楼层的 `⋯` 展开栏里有一个 🧭 图标：
- **AI 回复楼层**：显示本轮回复用到的路由情况（命中 / 关联 / 隐藏了哪些）。
- **玩家楼层**：显示该输入触发的路由是否有缓存 + 缓存内容，并可「清除本层缓存」。

## 依赖

- SillyTavern ≥ 1.16
- （可选）[ST-Prompt-Template](https://github.com/) 扩展：仅当你的世界书条目里用到 **EJS/宏** 时才需要——本插件保留 ST 原生注入流程，EJS 由它照常处理。

---

## 说明

- 副模型的 API Key 只存在各自 ST 的本地设置里，**从不进代码 / 导出文件**。
- 结果缓存是本会话内存，重载 ST 后清空（属正常）。
- 导入的旧楼层 / 启用本插件前的楼层没有路由记录，逐层面板会提示"无记录"。

## 开发

扩展即 `manifest.json` + `index.js` + `style.css`，无构建步骤，改完刷新 ST 页面即生效。
`main` 为发布分支（用户跟踪），`dev` 为开发分支。
