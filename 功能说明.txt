【规则路由 · Rule Router】—— SillyTavern 世界书条目智能路由扩展

━━━━━━━━━━━━━━━━━━━━
它解决什么问题？
━━━━━━━━━━━━━━━━━━━━
你的世界书里常有一堆「常驻（蓝灯）」条目——规则、设定、状态模板等，
它们每回合都被塞进提示词，既费 token，又会干扰主模型的注意力。

本插件让这些条目「按需激活」：每次生成前，用一个便宜快速的副模型（flash）
判断当前情境到底需要哪些条目，只把相关的交给主模型，其余本回合临时隐藏。

★ 零改卡：不改你世界书条目的开关、不写盘，隐藏只作用于「本次生成」。
★ 通用：任何角色卡都能用，逐条目在界面里配置触发条件。
★ 安全：没配置副模型 / 判定失败时，一切照常，不影响游玩。

***重要提示：因为插件有API调用，如果用公益站API可能会让你被封；用DSv4-flash就行了***
━━━━━━━━━━━━━━━━━━━━
下载 / 安装
━━━━━━━━━━━━━━━━━━━━
SillyTavern → 扩展 → 安装扩展 → 填入 git 地址：

    https://github.com/Awene/cultivation-rule-router

国内连不上 GitHub 就用镜像地址：

    https://gitclone.com/github.com/Awene/cultivation-rule-router

装好后：左下角魔棒菜单 → 「规则路由配置」进行设置。
（支持自动更新；需要 SillyTavern ≥ 1.16）

━━━━━━━━━━━━━━━━━━━━
主要功能
━━━━━━━━━━━━━━━━━━━━
· 逐条目「文字过滤」：给条目写一句启用条件，副模型据情境判断该不该开。
· 规则关联：A 触发时连带开启 B（支持 A→B、C→B 多源，及 A→B、B→A 互相关联）；被关联条目也可自设条件独立触发。
· 路由结果写入 chat 变量（命中/关联/隐藏），角色卡可用 EJS 或宏 {{getvar::路由隐藏规则}} 据此条件拼装提示词。
· 副模型可自由配置：接口地址 / Key / 获取模型 / 可编辑的路由提示词 + 预览。
· 上下文可调：思维链分隔符、收录最近几条 GM 回复、标签剔除、结果缓存条数。
· 结果缓存：重生成 / 回退同输入时复用旧判定，不重复调用副模型。
· 逐层查看：每层消息按钮里可查看本层的路由情况 / 缓存，并单独清除本层缓存。
· 配置导入 / 导出：方便分享给别人（导出绝不含 API Key）。
· 界面轻提示：每次生成实时显示「候选 / 命中 / 关联 / 隐藏」统计。
· 一键总开关。

━━━━━━━━━━━━━━━━━━━━
可选依赖
━━━━━━━━━━━━━━━━━━━━
若你的世界书条目里用到 EJS/宏，需要额外装 ST-Prompt-Template 扩展
（本插件保留 ST 原生注入流程，EJS 由它照常处理）；不用 EJS 则无需安装。

更新日志
━━━━━━━━━━━━━━━━━━━━
v0.11.3
· 修复上一版限高后「规则过滤」区被压成 0 高、正常尺寸下看不到规则列表（须缩放才可见）的问题：
  规则过滤区不再用 flex:1 撑满，改为自然高度、由整体面板滚动。
v0.11.2
· 修复手机等窄屏/矮屏下配置面板超出屏幕、顶部编辑框被顶出可视区且无法上滑的问题：
  面板自身限高并内部滚动（max-height + overflow），不再依赖弹窗撑高。
v0.11.0
· 过滤配置改为按「条目名称」存储（原按 uid）。tavern_sync push / 重新导入世界书会重新生成 uid，
  旧版会导致已配好的路由过滤全部失配、静默失效；改按名称后可跨 push / 跨导入稳定复用。
  含一次性自动迁移：首次加载时把旧的 uid 键配置按其记录的名称迁移过来（幂等，无需手动操作）。
v0.11.1
· 手动预览模式改为所见即所得：打勾=激活、不勾=隐藏，且无视关联触发（用户勾什么就是什么）。
v0.10.0
· 新增「手动预览模式」：开启后跳过 flash，仅用下方候选条目的勾选状态决定本回合激活/隐藏
  （仅已配路由的候选条目）写入路由变量。配合酒馆助手「提示词查看器」点刷新，即可预览该状态
  下的最终提示词，用于调试卡侧 EJS/宏条件（如 NSFW、各规则）而无需真实调用 flash。持久开关。
v0.9.x
· 路由结果写入 chat 变量（路由命中规则/路由关联规则/路由隐藏规则），
  角色卡可用 EJS 或宏据此条件拼装提示词
